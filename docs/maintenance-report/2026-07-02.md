## Code Quality

**Fixed (real correctness bugs, not style):**

1. **Unclosed `<div data-aos="zoom-in">` in the Projects section** (`index.html`, previously line 397) — every other top-level section wraps its content in a `data-aos="zoom-in"` div that is explicitly closed (e.g. `<!-- end s-testimonials -->`), but the Projects section's wrapper was never closed, which cascaded into a mismatched `</body>`/`</html>` at the end of the document. Added the missing `</div>`. Verified via `html-validate` (all `close-order` errors resolved) and a rendered-page screenshot showing the Projects grid is unaffected visually.
2. **Malformed `<head>` in `404.html`** — a `<style>` block was placed *before* the `<head>` tag opened, which caused the HTML5 parser to implicitly open a head element and then treat the explicit `<head>...</head>` as invalid nested content (browsers tolerate this, but it's invalid markup). Moved the `<style>` block inside `<head>`, after the existing stylesheet links.
3. **Unencoded `&` in visible text** (`index.html`, "HTML & CSS", "Google Sheets & Google Apps Script", etc.) — encoded as `&amp;` per the HTML spec.

**Not changed (reviewed, judged out of scope):**

- The vendored "Hola" template uses `<div>` elements inside `<h1>`/`<h3>` headings for text-flip/moving-letter animation effects, and self-closing void elements (`<br/>`), inline `style=""` attributes, and non-`type`-annotated buttons throughout. These are template-design choices used consistently across the whole file (100+ occurrences) that render correctly in all major browsers and are targeted by existing JS/CSS by class name, not by strict-HTML semantics. Rewriting them would be a large, high-risk, purely cosmetic change with no functional benefit, so they were intentionally left as-is and excluded from the CI HTML-validation rule set (see `.html-validate.json`) rather than silently ignored.
- `debug.log`-style junk files: only one instance existed; removed (see Security Review).

No dead code, duplicate logic, or unused dependencies were identified beyond the above.

## CI/CD

The repository previously had **no CI/CD** at all. Added `.github/workflows/ci.yml`, which on every push/PR to `main`:

1. Runs `scripts/check-local-links.js` — fails the build if any local `href`/`src` reference in `index.html`/`404.html` points to a non-existent file (would have caught, for example, a typo in the jQuery filename during this exact update).
2. Runs `html-validate` against `index.html` and `404.html` using a project-specific `.html-validate.json` config that keeps structural/correctness rules (unclosed tags, invalid nesting, unencoded entities, duplicate IDs, etc.) and disables rules that only flag the vendored template's intentional style choices (see Code Quality above).

Both checks currently pass cleanly against the updated code and gate merges to `main`.
